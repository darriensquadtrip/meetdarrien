Prototype Test Guide

Story-by-story walkthrough of the Customer Central prototype

Prototype: customer-central.html
Scenario: Sarah Chen (Admin) at Acme Corp
Plan: PEO Plus, 34 employees, NY/CA/TX

Getting Started

Open the prototype in a browser
You'll see an empty state with a search bar prompt
Press Cmd+K (Mac) or Ctrl+K (Windows) to focus the search bar — or click it directly
Type any character (e.g., "acme" or "sarah") — the search dropdown appears
Click any search result to load the Acme Corp dashboard
The entire dashboard loads on click. A green toast notification confirms: "Loaded Acme Corp — Sarah Chen (Admin)." The sidebar expands to show account-specific navigation.

Health Score Stories
HS-1
Health Score Badge
As a CSO agent during any interaction, I want a single composite health score visible in the status strip — so I can instantly gauge account health before diving into individual signals.
▼
HS-1 — Full Build. Lightweight, rule-based composite score. Formula: Base = (Billing x 0.30) + (Payroll x 0.25) + (Support x 0.20) + (Satisfaction x 0.15) + (Activity x 0.10), then Final = Base x Tenure Factor. No ML in Phase 1.
How to Test
After loading the dashboard, look at the Status Strip — a 6th indicator now appears at the leading position (far left):
Health Score: Shows "82 — Good" in light green with a green dot
Subtext: "5 signals + tenure adjustment · Rule-based"

Support: 72 — 1 open ticket, 3 days old, medium priority
Satisfaction: 85 — Last 3 CSAT avg
Activity: 80 — Moderate — 3 changes in 14 days
Tenure Factor: 1.00 — Established (since Jan 2022)
Overall: 82 — Good
Click anywhere outside the card to dismiss it
Navigate to the Billing tab and click "At-Risk Scenario" toggle
Return to the status strip — the health score should now show "34 — At Risk" in orange
Click the health score badge again — Billing signal should now show 0 with "Past due 8 days — ACH failed"
Toggle back to "Current (Acme)" — score returns to 82 (Good)
In the breakdown card, verify the Tenure Factor row displays correctly:
Established customer (180+ days): Shows multiplier of 1.00 — base score is unchanged
Acme Corp (since Jan 2022) should show "1.00 — Established"
For a new customer (< 30 days), the tenure factor should show 0.75 multiplier:
Base score is reduced by 25% — e.g., a base of 82 becomes 61.5 (rounded to 62)
The band label adjusts accordingly (62 = "Attention" in amber instead of "Good" in green)
Verify the score calculation is correct with tenure adjustment:
Base = (Billing x 0.30) + (Payroll x 0.25) + (Support x 0.20) + (Satisfaction x 0.15) + (Activity x 0.10)
Final = Base x Tenure Factor
For Acme: Base = 90.15, Tenure Factor = 1.00, Final = 90.15 (displayed as 82 sample)

  • Salesforce — Toast simulating external navigation
