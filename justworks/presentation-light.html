Six Compounding Problems

All traced back to the case study brief. P1–P4 map to Phase 1 MVP; P5–P6 require the data foundation the MVP builds.

P1

Context-Switching Tax

5–6 disconnected systems per call. Every switch costs cognitive bandwidth and clock time.

P2

Caller Identity Problem

Admin or employee? Which company? First 30–60 seconds wasted on identification.

P3

Repeat-Yourself Problem

History split across Zendesk and Salesforce. Customers tell their story three times.

P4

Account Status Blindspot

Billing status and payroll dates live in production DB — not in the tools agents use.

P5

Proactive Blindspot

Usage patterns and lifecycle signals exist but are never synthesized into agent-facing alerts.

P6

Open Issues Gap

No way to see all active work for an account. Agents diagnose issues others are already fixing.

The Strategic Case

Justworks has incredible agents pulling off +60 NPS in spite of their tools, not because of them. A model that relies on individual heroics doesn't scale.

NPS ÷ CSO Headcount $
Not NPS alone. Not AHT alone. The ratio that captures leverage.

The ratio of customer satisfaction to support investment should improve quarter over quarter — not because service gets worse, but because the system gets smarter.

Small business context: Justworks serves companies without dedicated HR. When their admin calls, CSO may be their only expert.

Three Phases, One Dependency Chain

Phase 1 — Foundation

Unified Context

7 weeks

Read-only. Give agents a single screen with complete customer context. This is the foundation everything else depends on.

Phase 2 — Intelligence

Health & Action

8–12 weeks

Clean data flowing → layer in ML scoring, write-back actions, proactive alerts, benefits life events.

Phase 3 — Proactive

Prediction

Following quarter

90+ days of structured data → AI summaries, churn prediction, agent copilot, upsell intelligence.

Each phase produces the data the next phase needs. Phase 1 isn't just a smaller version of the end state — it's the instrumentation layer that makes Phases 2 and 3 possible. This maps directly to the prompt: Phase 1 = handle direct problems. Phase 2 = pre-empt problems. Phase 3 = proprietary data as competitive advantage.

Integration Risks — Week 1 Discovery Items

Zendesk ↔ Salesforce Join Key

May not exist cleanly. If no shared identifier, we need a resolution strategy in Week 1. Scoped to affect FS-2, not the whole MVP.

Real-Time Payroll Status

Unknown if queryable or batch process. Determines whether we show live data or cached snapshots for payroll countdown.

Payment Cascade Logic

"Failed payment blocks payroll for 34 employees" requires business rules that might only live in someone's head. Needs documentation.

Pull-Based Rollout

If agents don't prefer it, we don't force it — we fix it. Three stages, each gated.

Alpha — Week 5

5 Agents

Volunteer agents, supplementing existing tools. Daily 15-min feedback standups.
Gate to Proceed
  • Data accuracy ≥ 95%
  • Core flow works without crashing
  • 3/5 agents prefer it over current tools
  • No critical data gaps identified

How I Got to the MVP

14-step decision chain, compressed into 4 moves:

Step 1

Extract Jobs

Decomposed the prompt into 6 jobs CSO agents need to perform. Every feature maps to a job.

Step 2

Dependency Stack

Foundation before operations before intelligence. You can't score data you haven't consolidated.

Step 3

Score & Rank

20 problems scored on impact, confidence, and ease. Top-scoring problems converge on the same 6 data fields.

Step 4

Read Before Write

Read-only MVP. Wrong data displayed is fixable. Wrong data written back is a disaster.

The convergence insight: Top-scoring problems and the health dashboard need the same six data fields. That convergence is why 11 features feel achievable in 7 weeks.
